About this extractor vision

A seventh extraction approach that hands each PDF page to Anthropic's Claude vision-language models (via OpenRouter) instead of a deterministic parsing library.

How it differs from v1–v6

Versions v1–v6 on the comparison page read PDFs with PyMuPDF, pdfplumber, unstructured.io, docling, pdfminer.six, and Camelot — fast, free, and fully reproducible. This page renders each page to an image and asks Claude to describe what it sees: useful for scanned, untagged, or visually complex PDFs where the structural libraries miss content.

What you get back

  • Per-page Markdown reconstructing the text in reading order, including headings and lists.
  • Tables as 2-D arrays. Cells the model wasn't sure about are flagged with a yellow border and a tooltip.
  • Figures as short natural-language descriptions of charts, diagrams, and images.
  • Usage & cost — input/output tokens and an estimated USD cost shown at the bottom of every run.

Choosing a model

  • Haiku — bulk runs, faster and cheaper. Good default for first passes.
  • Sonnet — slower and pricier, better at dense layouts, small fonts, and tricky tables.

Cost, caching & control

  • The Estimated cost panel in the sidebar shows a low–high range per model before you spend anything. After your first few runs, the estimate is calibrated from your own usage history (prefers PDFs of similar length).
  • Results are cached on disk under output/<pdf>/vision/. Re-selecting the same PDF + model loads instantly with no API call. Use Force to re-spend tokens.
  • Cancel a long run at any time — chunks already returned are saved as a partial result. Click Resume to continue from where it stopped without paying for completed chunks again.
Ready to try it? Pick a PDF on the left, choose a model, and click Run extraction. The first run on a new PDF takes roughly a few seconds per page.
